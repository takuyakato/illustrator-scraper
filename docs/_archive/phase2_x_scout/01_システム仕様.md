# Phase2-01: Xスカウト自動化 システム仕様

**バージョン**: 1.0
**作成日**: 2026-04-21
**ベース**: `仕様元/CLAUDE.md v0.1.0` を確定前提に合わせて更新

---

## このドキュメントの役割

Xフォローグラフから候補イラストレーターを自動発掘し、Googleスプレッドシートに投入するシステムの技術仕様。Claude Code実装担当が読む想定。

---

## 1. 目的

BLサンド・Capuri・BerryFeel・Webtoon の各プロジェクトで依頼可能なイラストレーター候補を継続的に発掘する。Xのフォローグラフを辿り、フィルタを通過した候補をスプレッドシート「候補プール」に流し込む。

---

## 2. アーキテクチャ

```
┌─────────────────────────────────────────────┐
│ illustrator-scraper（このプロジェクト）      │
│                                             │
│  ① Notion API                               │
│     マスターDBから「シード」アカウント抽出   │
│     (条件: ランクS/A かつ マスターステータス │
│             = 依頼成功)                     │
│                                             │
│  ② Browser Use 2.0                          │
│     各シードのフォロー一覧取得              │
│                                             │
│  ③ フィルタリング (TypeScript)              │
│     プロフィール・フォロワー数・重複除外     │
│                                             │
│  ④ Google Sheets API                        │
│     候補プールシートに書き込み              │
└─────────────────────────────────────────────┘
                     │
                     ▼
         ┌──────────────────────┐
         │ スカウトがGSheet上で   │
         │ 判定 → 「転記」ボタン │
         └──────────┬───────────┘
                    ▼
         ┌──────────────────────┐
         │ GAS → Notion API     │
         │ ○判定のみ転記         │
         └──────────────────────┘
```

本プロジェクトは **①〜④** を担当。GAS部分は `gas-illustrator-sync`（別管理）。

## 6. Phase別実装計画

### Phase 0: セットアップ（1日）

- pnpm init、TypeScript (ESM)、ESLint + Prettier、vitest セットアップ
- `.env.example` 作成・認証情報取得・`.env` 作成
- Notion API 疎通確認
- Google Sheets API 疎通確認

### Phase 1: Notion シード抽出（1日）

**目的**: マスターDBから「ランクS/A かつ マスターステータス=依頼成功」の作家のXアカウント一覧を取得。

- `src/notion/client.ts`: Notion SDK ラッパー
- `src/notion/seed-fetcher.ts`:
  - フィルタ: `ランク IN (S, A)` AND `マスターステータス = 依頼成功`
  - 取得プロパティ: `作家名`, `Xリンク`, `ランク`, `TL適性`
- `Xリンク` から username を抽出（例: `https://x.com/username` → `username`）
- `Xリンク` が空のレコードは除外
- `ランク=S` は priority=high、`ランク=A` は priority=normal
- `scripts/seed-dry-run.ts` で動作確認

**出力型**:
```typescript
interface SeedAccount {
  notionPageId: string;
  artistName: string;
  xUsername: string;       // "@" なし
  powerRank: 'S' | 'A';
  priority: 'high' | 'normal';
}
```

### Phase 2: Browser Use 2.0 でフォロー一覧取得（3〜5日）

- `src/scraper/browser.ts`: Browser Use 2.0 初期化
- X ログインフロー
  - セッションCookie保存・再利用
  - CAPTCHA検出時はログを残して処理中断
- `src/scraper/follow-list.ts`:
  - `https://x.com/{username}/following` にアクセス
  - 無限スクロールで最大 `SCRAPER_MAX_FOLLOWS_PER_SEED` 件取得
  - username / displayName / bio / followerCount を抽出
- リクエスト間ランダム遅延
- 凍結・レート制限検出時は指数バックオフ

### Phase 3: フィルタリング（2日）

- `src/filter/profile-keyword.ts`
  - 通過キーワード（日本語）: `イラスト`, `絵`, `漫画`, `デザイン`, `絵描き`, `お絵かき`
  - 通過キーワード（英語）: `illustration`, `artist`, `illustrator`, `drawing`
  - 除外キーワード: `AI`, `生成AI`, `stable diffusion`, `midjourney`
  - bio 空は保留
- `src/filter/follower-count.ts`: FILTER_MIN_FOLLOWERS 〜 FILTER_MAX_FOLLOWERS
- `src/filter/post-frequency.ts`: 直近30日で FILTER_MIN_POSTS_LAST_30D 件以上
- `src/filter/duplicate.ts`:
  - マスターDBの `Xリンク` と突合
  - スプレッドシート既存データと突合
  - 重複は除外（フラグは残す）

**出力型**:
```typescript
interface Candidate {
  xUsername: string;
  displayName: string;
  bio: string;
  followerCount: number;
  followingCount: number;
  postsLast30d: number;
  pixivUrl?: string;
  portfolioUrl?: string;
  seedAccount: string;
  detectedAt: Date;
  isDuplicateInMaster: boolean;
  filterScore: number;  // 0-100
}
```

### Phase 4: Google Sheets 書き込み（1日）

- `src/sheets/client.ts`: Sheets API 初期化
- `src/sheets/writer.ts`: 候補プールに追記
- 書き込み前にスプレッドシート既存Xアカウントを取得して重複回避
- バッチ書き込み（1回で複数行）
- 列仕様は `02_スプレッドシート仕様.md` を参照

### Phase 5: バッチ実行（1日）

- `src/index.ts`: オーケストレーション
- `scripts/local-batch.ts`: ローカル実行
- 当面はローカル手動実行（`pnpm run batch`）で運用
- GitHub Actions 移行は運用安定後

## 8. エラーハンドリング方針

### 致命的エラー（処理中断）

- X アカウントログイン失敗 → 即時通知
- Notion API 認証エラー → 処理中断
- Sheets API クォータ超過 → 6時間後再試行

### 回復可能エラー（ログして継続）

- 特定シードのフォロー一覧取得失敗 → スキップ
- 特定候補のプロフィール取得失敗 → その候補をスキップ
- 一時的ネットワークエラー → 最大3回リトライ

### ロギング方針

pinoで構造化ログ。例：
```typescript
logger.info({
  phase: 'scraping',
  seed: 'example_account',
  candidatesFound: 42,
  durationMs: 12345
}, 'シードの処理完了');
```

## 9. 開発時の注意事項

### Claude Code使用時

1. このファイルを読んでから作業開始
2. Phase単位で実装し、各Phase完了時に動作確認

## 10. トラブルシューティング

### Xログイン失敗

- 2FA一時OFF、またはTOTP自動入力を追加
- セッションCookieが古い場合は削除して再ログイン
- IPブロックの可能性 → VPN経由で接続

### Notion API 404

- インテグレーションがDBに接続されているか確認
- `NOTION_MASTER_DB_ID` 確認
- プロパティ名が変更されていないか確認

### Sheets permission denied

- サービスアカウントがスプレッドシートに「編集者」共有されているか確認

### フィルタ通過率が低い

- `FILTER_MIN_FOLLOWERS` を下げる
- キーワードリストを拡張
- 除外キーワードが厳しすぎないか確認
