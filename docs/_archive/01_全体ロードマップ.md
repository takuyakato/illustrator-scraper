## 背景と目的

### 現状の課題

- プロデューサーがX（旧Twitter）の既存フォロー欄を手作業で1つずつ確認しており、工数が膨大
- Notion「イラストレーター情報」DBが肥大化・冗長化しており、候補探索段階でNotionを開くと重い
- 複数プロジェクト（BLサンド・Capuri・BerryFeel・Webtoon）で探索ルールがばらついている

### ゴール

- **候補探索 → スカウト判定 → Notion転記** の3段階を分離し、各段階に最適なツールを充てる
- Notion DBは「本当に依頼対象になり得る人だけ」が入るプールにし、運用を軽くする
- 週次で継続的に新規候補が供給される状態を作る

### 理想フロー

```
┌─────────────────────────────────────────────┐
│ ① Claude Code バッチ（夜間・自動）            │
│    既存DBのシード → Xフォロー取得 → フィルタ   │
│    → Googleスプレッドシートに追加              │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│ ② スカウト（週50件・GSheetのみで完結）         │
│    ○/保留/× 判定・画力仮ランク・コメント       │
│    「📤 Notionへ転記」ボタン押下              │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│ ③ GAS（自動）                                 │
│    ○判定のみNotionに登録（マスターステータス=候補）│
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│ ④ オーナー（Notion・随時）                    │
│    画力ランク確定・TL適性・プロジェクト適性    │
│    → マスターステータス「依頼可能」            │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│ ⑤ 連絡担当（Notion・随時）                    │
│    依頼連絡 → 依頼成功 / 多忙辞退 / 連絡不通   │
└─────────────────────────────────────────────┘
```

## 2段階で進める

### Phase 1: Notion DB整理（先行）

**目的**: スクレイピング側が想定するDBスキーマ・運用ルールを実DBに反映する。

**アウトプット**:

- 新プロパティ追加（マスターステータス・TL適性・検出元アカウント等）
- 冗長プロパティ整理（ステータス類3種の統合、メール重複解消）
- Berryfeel別DBのメインDBへのマージ
- 既存レコードのステータス移行
- Notionインテグレーション接続（Claude.Bisque）

**関連ドキュメント**:

- `phase1_notion_db/01_現状分析とギャップ.md`
- `phase1_notion_db/02_DB再設計仕様.md`
- `phase1_notion_db/03_マイグレーション手順.md`

**期間目安**: 3〜5営業日（手作業中心）

---

### Phase 2: Xスカウト自動化（Phase1完了後）

**目的**: 整理されたDBをシードとして、Xフォローグラフからの候補発掘を自動化する。

**アウトプット**:

- `illustrator-scraper` リポジトリの実装（TypeScript / Browser Use 2.0）
- Googleスプレッドシート「候補プール」の構築
- GASによるNotion転記機能
- ローカル手動実行での運用開始

**関連ドキュメント**:

- `phase2_x_scout/01_システム仕様.md`
- `phase2_x_scout/02_スプレッドシート仕様.md`
- `phase2_x_scout/03_準備タスク.md`

**期間目安**: 実装2週間＋試運転1週間

---

### 並行: 運用・周知ドキュメント

Phase1完了時点で、運用ドキュメントを整備してチームへ周知する。

**関連ドキュメント**:

- `operation/02_スカウト向けマニュアル.md`
