# イラストレーター情報データベースの使い方

**Notion版（チーム共有用）**: <https://app.notion.com/p/35ec50637a7b80319088c8c2a9635f7a>

「イラストレーター情報DB」は、**Berryfeel・Capuri・BLサンド の各案件で起用するイラストレーター候補を一元管理する Notion データベース**です。

このページでは、

- 実際に **連絡してステータスを管理する** とき
- DB に **新しいイラストレーターを追加する** とき

の2つの場面で、何をどう操作すればよいかをまとめます。

新規追加のときだけ、補助ツールとして Googleスプレッドシート「illustrator-scraper 候補プール」を併用します。

### 3つのツールの関係

このシステムは **Supabase（データベース）・Notion・Googleスプレッドシート「illustrator-scraper 候補プール」** の3つで動いています。

- **Supabase（データベース）**：すべてのデータを保管する裏側の中心。人は直接触りません。
- **Notion「イラストレーター情報DB」**：確定したイラストレーターの **連絡・ステータス** を人が管理する場所。
- **Googleスプレッドシート「illustrator-scraper 候補プール」**：自動で集めた候補を、人が **「イラストレーターか否か」を選別する** 場所。

NotionとGoogleスプレッドシートは直接はつながっておらず、**どちらも Supabase を介して自動同期** します（反映は最大10分）。そのため、片方で編集した内容は自動でデータベースに集約され、必要な範囲でもう片方にも行き渡ります。

```
                自動収集（スクレイパー）
                        │
                        ▼
          ┌───────────────────────────┐
          │   Supabase（データベース）   │ ← 全データを保管する中心。人は触らない
          └───────────────────────────┘
               ▲                      ▲
        最大10分で自動同期       最大10分で自動同期
               ▼                      ▼
 ┌────────────────────┐  ┌──────────────────────────────┐
 │ Notion              │  │ Googleスプレッドシート          │
 │ イラストレーター情報DB │  │「illustrator-scraper 候補プール」│
 │                     │  │                              │
 │ 確定した人の          │  │ 集めた候補を人が              │
 │ 連絡・ステータスを管理  │  │「イラストを描く人か」を選別     │
 └────────────────────┘  └──────────────────────────────┘
```

## 1. ランクの考え方

ランクは **「BL / TL などの女性向けイラストを依頼したときに、女性に刺さる画力かどうか」** で付けます。
線の繊細さ、目の描き方、配色、雰囲気作りなどを総合した、女性向け作品での訴求力評価です。

### ランクの定義

| ランク | 意味 |
|---|---|
| **S** | 女性に強く刺さる画力。最上位。 |
| **A** | 女性に刺さる画力。主力。 |
| **B** | 一定の画力。 |
| **C** | 弊社では依頼しない。 |

### レーベル別の起用ランク

| レーベル | ジャンル | 起用するランク |
|---|---|---|
| **Berryfeel** | TL | S / A |
| **Capuri** | BL | S / A / B |
| **BLサンド** | BL（YouTubeチャンネル） | A / B |

逆引きすると、

- **Sランクの作家** → Berryfeel または Capuri に起用候補
- **Aランクの作家** → Berryfeel / Capuri / BLサンド すべて起用候補
- **Bランクの作家** → Capuri または BLサンド に起用候補
- **Cランクの作家** → 起用しない（DBには残るが連絡対象外）

ランクは Notion 上でプロデューサーが調整できます。実際にやり取りしてみて印象が変わったら更新してください。

## 2. 連絡するとき

連絡したら、**必ず3項目をセットで更新** します。

### 3点セット（必須）

| プロパティ | 入力内容 |
|---|---|
| **マスターステータス** | `候補` → `連絡中` に変更 |
| **連絡した日** | 連絡した当日の日付 |
| **連絡した人** | 自分の名前 |

この3つが揃っていないと、14日ルール（後述）と履歴管理が正しく動きません。

### 再度連絡する日のルール（自動遷移）

`再度連絡する日` が今日以前になると、毎日深夜3時に自動で `再連絡` に切り替わります。
元のマスターステータスは問いません。`多忙辞退` だけでなく、`返信なし` や `条件次第` などでも、日付が到来すれば `再連絡` になります。
備考欄にも `[自動遷移: 再連絡]` が追記されます。

### 14日ルール（自動遷移）

`連絡中` のまま `連絡した日` から14日が経つと、毎日深夜3時に自動で `返信なし` に切り替わります。
備考欄にも `[自動遷移: 連絡中→返信なし]` が追記されます。
人手で戻すことも可能です。

## 3. 候補プールから新しいイラストレーターを追加するとき

**候補プール（Googleスプレッドシート）**：<https://docs.google.com/spreadsheets/d/1QkjdYjbVXedRYSjd-D2PREKYfYlAGw7LNeIptYbiHa8/edit?usp=sharing>

候補プールは、illustrator-scraper が自動で集めた「イラストレーターになりそうなXアカウント」が一覧で並ぶGoogleスプレッドシートです。
ここで人の目で「イラストレーターかどうか」を選別し、起用候補をDB（Notion）に新規追加していきます。

### 候補プールに溜まる仕組み

- DB に登録済みの **S / A ランクのイラストレーターがフォローしている人** を、システムが週1回取得
- そのうち **プロフィールに Pixiv または ポートフォリオサイトのリンクがある人** だけが Googleスプレッドシート「候補プール」に並ぶ

ただしこの段階では絵を描く人かどうかは確定していません。
**AI絵師・漫画家・声優・編集者・ファンアカウントなども混ざります。**
だからまず「イラストレーターか否か」を判定するのがこの作業の役割です。

### 補充の2ステップ

```
[ステップ1] イラストレーターか判定する
              │
              ├─ NO  → J列に「イラストレーターじゃない」
              │        → 候補プールから消える
              │
              └─ YES → [ステップ2] 仮ランクを付ける
                        → K列に S / A / B / C
                        → 最大10分で Notion DB に追加される
```

### 入力する列（J / K / L / M）

| 列 | 入力 | 説明 |
|---|---|---|
| **J（判定）** | `イラストレーターじゃない` または 空 | イラストレーターでなかったら入力 |
| **K（仮ランク）** | `S` / `A` / `B` / `C` または 空 | イラストレーターだったら入力。1. のランク定義に従う |
| **L（確認者）** | `北條` / `三村` / `加藤` など | 複数人なら `北條, 三村` のように入力 |
| **M（コメント）** | 自由記述 | 引き継ぎメモなど |

### 触ってはいけないところ

- **A〜I 列**（検出日・Xアカウント・表示名・プロフィール 等は自動入力）
  - 特に **C列（Xアカウント）** を書き換えると同期が壊れます
- **行の削除**（不要な行はフィルタで非表示にする）

### N列（同期状態）

| 値 | 意味 | 対応 |
|---|---|---|
| 未同期 | これから反映される | 放置でOK |
| 同期済み | Notion に反映済み | 放置でOK |
| 同期失敗 | エラーで反映できなかった | 加藤さんに連絡 |

判定をやり直したい場合は、**N列を手動で `未同期` に書き換える** と次のサイクルで再反映されます。

## 補足

### 反映タイミング早見表

| 操作 | 反映先 | 所要時間 |
|---|---|---|
| Googleスプレッドシート で仮ランク（K列）入力 | Notion DB に新規追加 | 最大10分 |
| Googleスプレッドシート で「イラストレーターじゃない」（J列）入力 | 候補プールから消える | 最大10分 |
| Notion でマスターステータス・連絡情報を更新 | システム内部に反映 | 最大10分 |
| `連絡中` → `返信なし`（14日ルール） | 自動遷移 | 毎日深夜3時 |

### Notion で編集しても自動で戻る項目

以下は Googleスプレッドシート / スクレイパー側が正なので、Notion で書き換えても次の同期で元に戻ります。

- 作家名
- Xリンク
- フォロワー数
- 検出元
- 初回検出日時
- 最終検出日時

修正したい場合は加藤さんに連絡してください。

### 困ったとき

| 症状 | 対応 |
|---|---|
| Googleスプレッドシート の N列が「同期失敗」 | 加藤さんに連絡 |
| Googleスプレッドシート で仮ランクを入れたのに10分経ってもNotionに出ない | N列が「同期済み」になっているか確認。なっていなければ加藤さんに連絡 |
| Notion で更新したのに反映されない感じ | 10分待っても直らなければ加藤さんに連絡 |
| いつもと違う動き | 加藤さんに連絡。Notion の「同期失敗ログ」ページに手がかりが残っている場合があります |
